# 鼓励储蓄罐 - 部署教程

## 方案一：Vercel 部署（推荐）

### 前置条件
- 一个 GitHub 账号（https://github.com）
- 项目代码已准备好

---

### 第一步：安装 Git 并初始化仓库

如果项目还没有初始化 Git，打开终端执行：

```bash
cd d:\project\mypoint
git init
git add .
git commit -m "初始提交"
```

---

### 第二步：创建 GitHub 仓库并推送代码

1. 打开 https://github.com/new
2. 填写仓库信息：
   - **Repository name**：`mypoint`
   - **Description**：鼓励储蓄罐 - 儿童积分管理系统
   - **可见性**：选择 Private（私有）或 Public（公开）
3. 点击 **Create repository**
4. 在终端执行推送命令：

```bash
git remote add origin https://github.com/你的用户名/mypoint.git
git branch -M main
git push -u origin main
```

> 如果提示登录，按提示输入 GitHub 用户名和 Personal Access Token（不是密码）。
> Token 生成方式：GitHub → Settings → Developer settings → Personal access tokens → Generate new token

---

### 第三步：添加 SPA 路由配置

项目使用了 React Router 客户端路由，需要添加重写规则，否则刷新页面会 404。

在项目根目录创建 `vercel.json` 文件：

```json
{
  "rewrites": [{ "source": "/(.*)", "destination": "/index.html" }]
}
```

提交并推送：

```bash
git add vercel.json
git commit -m "添加 Vercel 路由配置"
git push
```

---

### 第四步：注册并登录 Vercel

1. 打开 https://vercel.com
2. 点击 **Sign Up**
3. 选择 **Continue with GitHub**，用 GitHub 账号登录
4. 授权 Vercel 访问你的 GitHub 仓库

---

### 第五步：导入项目

1. 登录后进入 Vercel Dashboard
2. 点击 **Add New...** → **Project**
3. 在列表中找到 `mypoint` 仓库
4. 点击 **Import**

---

### 第六步：配置构建设置

在 Import Project 页面：

| 配置项 | 填写内容 |
|--------|---------|
| **Framework Preset** | 选择 `Vite` |
| **Root Directory** | `./`（默认即可） |
| **Build Command** | `npm run build`（自动填充） |
| **Output Directory** | `dist`（自动填充） |
| **Install Command** | `npm install`（自动填充） |

其他选项保持默认，点击 **Deploy** 按钮。

---

### 第七步：等待部署完成

- 部署通常需要 1-2 分钟
- 看到 🎉 庆祝画面表示部署成功
- Vercel 会自动分配一个域名，格式为：`mypoint-xxx.vercel.app`

---

### 第八步：访问你的应用

部署成功后：

1. 点击 Dashboard 中的项目名称
2. 点击域名链接即可访问
3. 也可以点击 **Visit** 按钮直接打开

---

### 后续更新部署

以后每次修改代码后，只需：

```bash
git add .
git commit -m "更新说明"
git push
```

Vercel 会自动检测到代码变更并重新部署，通常 1 分钟内完成。

---

### 自定义域名（可选）

1. 在 Vercel 项目页面 → **Settings** → **Domains**
2. 输入你的域名，如 `mypoint.yourdomain.com`
3. 按提示到域名服务商添加 CNAME 记录：
   - **类型**：CNAME
   - **名称**：mypoint
   - **值**：cname.vercel-dns.com
4. 等待 DNS 生效（通常几分钟到几小时）
5. Vercel 自动配置 HTTPS 证书

---

## 方案二：Cloudflare Pages 部署

### 前置条件
- 一个 GitHub 账号（https://github.com）
- 项目代码已推送到 GitHub（同方案一的第一、二步）
---

### 第一步：添加 SPA 路由配置

在项目的 `public` 目录下创建 `_redirects` 文件（注意没有扩展名）：

```
/*  /index.html  200
```

提交并推送：

```bash
git add public/_redirects
git commit -m "添加 Cloudflare Pages 路由配置"
git push
```

---

### 第二步：注册并登录 Cloudflare

1. 打开 https://dash.cloudflare.com/sign-up
2. 填写邮箱和密码注册
3. 到邮箱中验证邮箱地址
4. 登录 Cloudflare Dashboard

---

### 第三步：创建 Pages 项目

1. 在 Dashboard 左侧菜单点击 **Workers 和 Pages**
2. 点击 **Create** 按钮
3. 选择 **Pages** 标签页
4. 点击 **Connect to Git**

---

### 第四步：连接 GitHub 仓库

1. 首次使用需要授权：点击 **Connect GitHub**
2. 选择授权方式：
   - **All repositories**：授权所有仓库
   - **Only select repositories**：只授权指定仓库（推荐）
3. 如果选择指定仓库，勾选 `mypoint`
4. 点击 **Install & Authorize**
5. 回到 Cloudflare 页面，选择 `mypoint` 仓库
6. 点击 **Begin setup**

---

### 第五步：配置构建设置

在构建配置页面：

| 配置项 | 填写内容 |
|--------|---------|
| **Project name** | `mypoint`（或自定义） |
| **Production branch** | `main` |
| **Framework preset** | 选择 `Vite` |
| **Build command** | `npm run build` |
| **Build output directory** | `dist` |

其他选项保持默认，点击 **Save and Deploy**。

---

### 第六步：等待部署完成

- 首次部署需要 2-3 分钟
- 看到 **Success!** 表示部署成功
- Cloudflare 会自动分配域名，格式为：`mypoint.pages.dev`

---

### 第七步：访问你的应用

1. 在项目页面点击 **Visit site**
2. 或直接访问 `https://mypoint.pages.dev`

---

### 后续更新部署

与 Vercel 相同，推送代码即可自动部署：

```bash
git add .
git commit -m "更新说明"
git push
```

---

### 自定义域名（可选）

1. 在 Cloudflare Pages 项目 → **Custom domains** 标签
2. 点击 **Set up a custom domain**
3. 输入你的域名，如 `mypoint.yourdomain.com`
4. 如果域名已在 Cloudflare 管理：
   - Cloudflare 自动添加 DNS 记录，点击 **Activate domain**
5. 如果域名不在 Cloudflare：
   - 需要到域名服务商添加 CNAME 记录：
     - **类型**：CNAME
     - **名称**：mypoint
     - **值**：mypoint.pages.dev
   - 等待 DNS 生效
6. Cloudflare 自动配置 HTTPS 证书

---

## 两种方案对比

| 对比项 | Vercel | Cloudflare Pages |
|--------|--------|-----------------|
| **国内访问速度** | 一般（服务器在海外） | 较快（国内有 CDN 节点） |
| **免费额度** | 100GB 带宽/月 | 无限带宽 |
| **构建时间** | 6000 分钟/月 | 500 次/月 |
| **自动 HTTPS** | ✅ | ✅ |
| **自定义域名** | ✅ | ✅ |
| **部署速度** | 1-2 分钟 | 2-3 分钟 |
| **推荐场景** | 海外用户 | 国内用户（推荐） |

## 常见问题

### Q1：部署后刷新页面出现 404
**原因**：缺少 SPA 路由配置。
**解决**：确保已添加 `vercel.json`（Vercel）或 `public/_redirects`（Cloudflare Pages）。

### Q2：推送代码后没有自动部署
**原因**：可能没有正确连接 GitHub 仓库。
**解决**：在 Vercel/Cloudflare 的项目设置中检查 Git 集成状态。

### Q3：构建失败，提示依赖错误
**原因**：本地和线上 Node 版本不一致。
**解决**：在项目根目录创建 `.nvmrc` 文件，写入 `20`，提交推送。

### Q4：数据丢失了怎么办
**原因**：应用使用 LocalStorage 存储数据，数据绑定在浏览器中。
**解决**：清除浏览器缓存会导致数据丢失，建议不要清除应用数据。后续版本可考虑接入云同步。

### Q5：手机上如何添加到桌面
**iOS**：在 Safari 中打开应用 → 点击分享按钮 → 添加到主屏幕
**Android**：在 Chrome 中打开应用 → 点击菜单 → 添加到主屏幕
